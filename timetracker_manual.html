TimeTracker 自動登録スクリプト 使い方

日報の ＜業務計画＞ セクションを読み取り、TimeTrackerNX に実績を自動登録するスクリプトです。

📁 ファイル構成

ファイル役割
tt.ps1ワンコマンド実行スクリプト（日常使いはこれ）
tt_apikey.txtAPIキー保存ファイル（初回のみ編集）
timetracker_register.pyメインスクリプト（日報パース → API登録）
timetracker_config.json設定ファイル（ここを編集して分類ルールを変更）
output/tt_delete_date.py指定日の実績を全削除するユーティリティ

🚀 日常の使い方（tt.ps1）

tt.ps1 はAPIキーセット・登録実行をワンコマンドにまとめたスクリプトです。

初回セットアップ（一度だけ）

tt_apikey.txt の1行目にAPIキーを書いておくだけ。以降は自動で読み込まれます。

# tt_apikey.txt の中身（例）
e03bc76e-2f30-4edd-bd45-7ab44a55282d
APIキーを再生成したときは tt_apikey.txt の1行だけ書き換えれば OK。

コマンド一覧

コマンド動作
.\tt.ps1本日の日報を登録（APIキー自動セット → 即 EXECUTE）
.\tt.ps1 260624日付指定して登録（YYMMDD形式）
.\tt.ps1 260624 -Drydry-run（APIは叩かず分類結果だけ確認）
注意： --dry（ダブルダッシュ）は PowerShell では認識されません。
必ず -Dry（シングルダッシュ）を使ってください。

重複エラーが出た場合（再登録）

# 既存データを削除してから再登録
python output\tt_delete_date.py 2026-06-24
.\tt.ps1 260624

⚡ 直接 Python で実行する場合

① APIキーをセット

$env:TT_API_KEY = "your-api-key-here"
tt.ps1 を使う場合はこの手順は不要です。

② dry-run（内容確認）

# Outlookソース（デフォルト）
python timetracker_register.py --date 260623

# 日報ソース
python timetracker_register.py --date 260623 --source diary

③ 実際に登録

# Outlookソース（デフォルト）
python timetracker_register.py --date 260623 --execute

# 日報ソース
python timetracker_register.py --date 260623 --source diary --execute
登録済みの日付に再実行すると 重複エラー になります。先に削除してから再登録してください（後述）。

🗑️ 登録済みデータの削除

# 2026-06-22 の実績を全削除
python output\tt_delete_date.py 2026-06-22
削除後は再度 .\tt.ps1 または --execute で登録できます。

🔀 分類ロジック（2段階マッピング）

活動内容を プロジェクト と 作業種別 の2段階で分類し、workItemId を決定します。

活動テキスト
  ↓ project_rules で評価（上から順）
プロジェクトコード (JPQG1275 など)
  ↓ worktype_rules で評価（上から順）
作業種別 (会議・打合せ / 設計業務 など)
  ↓ workitem_map[プロジェクト][作業種別]
workItemId → TimeTrackerNX に POST

⚙️ 設定ファイル (timetracker_config.json)

project_rules — キーワード → プロジェクトコード

{
  "project_rules": [
    { "pattern": "DCAP",                       "project": "JPQG1275" },
    { "pattern": "FCM|TCM",                    "project": "JPQG1251" },
    { "pattern": "SCM|側方|機能安全|ISP|PAVCT", "project": "JPQG1252" },
    { "pattern": "PVM",                         "project": "JPQH0531" }
  ],
  "default_project": "JPRF0208"   ← どれにもマッチしない場合
}
パターンは Python 正規表現。| で OR 指定。上から順に評価され、最初にマッチしたルールを使います。

worktype_rules — キーワード → 作業種別

{
  "worktype_rules": [
    { "pattern": "レビュー|会議|定例|MTG|段コミ|打ち合わせ|打合せ|ヒアリング|相談|Weekly|DR",
      "worktype": "会議・打合せ" },
    { "pattern": "段取り|スケジュール|管理|TimeTracker|日報",
      "worktype": "プロジェクト管理" }
  ],
  "default_worktype": "設計業務"   ← どれにもマッチしない場合
}

workitem_map — プロジェクト × 作業種別 → workItemId

プロジェクト会議・打合せプロジェクト管理設計業務試作・製品評価その他
JPQG1275（DCAP）7381773818738207382173822
JPQG1251（FCM/TCM）7379973800738027380373804
JPQG1252（側方/機能安全）7380873809738117381273813
JPQH0531（PVM）106617106618106620106621—
JPRF0208（デフォルト）36752—→その他—36755
JPRF0208 に「設計業務」「プロジェクト管理」は存在しないため、fallback_worktype: "その他" (36755) が使われます。

✏️ ルールのカスタマイズ例

「段コミ」を BEV-S3前方 (JPQG1251) に振りたい

段コミはBEV-S3前方の会議だが、現在はプロジェクトキーワードがないため JPRF0208 に分類される。
project_rules に追加するだけで修正できる：

"project_rules": [
  { "pattern": "DCAP",                       "project": "JPQG1275" },
  { "pattern": "FCM|TCM",                    "project": "JPQG1251" },
  { "pattern": "SCM|側方|機能安全|ISP|PAVCT", "project": "JPQG1252" },
  { "pattern": "PVM",                         "project": "JPQH0531" },
  { "pattern": "段コミ|課MTG",               "project": "JPQG1251" }  ← 追加
]

「OrinSim」を DCAP プロジェクトに振りたい

{ "pattern": "OrinSim|確からしさ",           "project": "JPQG1275" }

新しいプロジェクトを追加したい

  1. project_rules にキーワードとプロジェクトコードを追加
  2. workitem_map に対応するプロジェクトコードと workItemId を追加
"workitem_map": {
  "JPXX0000": {
    "_project_name": "新プロジェクト名（メモ用）",
    "会議・打合せ":     "XXXXX",
    "プロジェクト管理": "XXXXX",
    "設計業務":         "XXXXX"
  }
}

📋 日報フォーマット（パース対象）

＜業務計画＞
06:30 ＜DCAP仕様書作業＞
10:00 ＜【定例】段コミ＞
11:00 ＜DCAP設計＞
13:00 ＜昼休み＞
14:00 ＜FCM機能確認＞
＜業務実績＞
  • 時刻は HH:MM 形式で行頭に書く（00:00〜23:59）
  • 活動内容は ＜　＞（全角かっこ）で囲む
  • ＜業務計画＞ と ＜業務実績＞ の間だけがパース対象
  • 「昼休み」「移動」などの skip_patterns にマッチする活動は自動スキップ
skip_patterns 一覧（クリックで展開） 
^$           （空行）
^昼休み
^業務終了
業務開始
移動
午後休
有休
～昼休憩～

timetracker_config.json の skip_patterns を編集して変更できます。

🔧 トラブルシューティング

エラー原因対処
環境変数 TT_API_KEY が未設定 APIキーがセットされていない $env:TT_API_KEY = "your-key" を実行
HTTP 400: TimeEntryOverlapped 同じ時間帯に既存の実績がある tt_delete_date.py で削除してから再登録
workItemId 未設定 workitem_map に該当エントリがない config の workitem_map を確認・追加
ModuleNotFoundError: requests requestsライブラリ未インストール .venv\Scripts\pip.exe install requests
＜業務計画＞セクションが見つかりません 日報の書式が違う ファイルに ＜業務計画＞ と ＜業務実績＞ があるか確認
